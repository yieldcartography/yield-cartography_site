# yieldcartography.com — site source

Static-HTML site deployed via Cloudflare Pages from this GitHub repo.

## Structure

```
.
├── README.md           # this file
├── build.py            # regenerate dashboard JSON from YIELDS CSVs
├── assets/             # source assets (logo, etc.)
└── dist/               # public-facing files served by Cloudflare Pages
    ├── index.html
    ├── about/index.html
    ├── curves/index.html         # fetches /data/yields.json on load
    ├── term-premia/index.html    # fetches /data/yields.json on load
    ├── _headers
    ├── _redirects
    ├── assets/
    │   ├── logo.svg
    │   └── style.css
    └── data/
        └── yields.json           # generated by build.py, ~1 MB
```

The dashboards are thin: HTML + Plotly fetched from CDN + a single async fetch
of /data/yields.json. All snapshot bond panels, forwards, SPF paths, and the
US/EA cross-country panel live inside that one JSON.

## Update workflow (after new YIELDS data)

```bash
cd ~/Documents/YC_site
python build.py
git add -A
git commit -m "Daily refresh $(date +%Y-%m-%d)"
git push
```

Cloudflare Pages auto-deploys within 30 to 60 seconds.

## Local preview

```bash
cd ~/Documents/YC_site/dist
python -m http.server 8000
# open http://localhost:8000 in browser
```
